Performance

In 2018, a good website should have a loading time of 2-3 seconds

Here, Loading time means that user has something useful to see or interact with

Statistics say that 53% users leave a website if pages take longer than 3 seconds

Think about performance, whenever you start to code something

All the giants in the field are chasing for performance

Images

Images are the No.1 fat candiates

JPEG, PNG, WEBP

Images add more weight to an application in most cases

Even though, images are not a render blocking asset. It is crucial to optimize them to save bytes over the wire

Images are the easiest assets to deal with. Know fundamentals, find some good compression tool. You're done.

Image Fundamentals

  • Is the image transparent? then PNG is must

  • Is the image has less colors? PNG works better

  • Is the image not transparent & has more colors? go for JPEG

  • Compression is must for all images

  • Don't commit unoptimized image to git repo

  • Make sure optimization doesn't reduce image quality visually

  • Don't compress the compressed. Always compress original image

There might be a confusion, which format to use for an image

Choosing the right format for the image is important
